# Claude Code チュートリアル 構想

## 目的

Claude Code を使ったことがない〜使い始めたばかりの開発者が、実際の開発業務で活用できるレベルになることを目指す。
「コマンドの説明」ではなく「どう使えば仕事が速くなるか」を軸に据える。

---

## ターゲット読者

- VSCode などのエディタで日常的に開発している人
- Claude（Web版）は使ったことがあるが、CLI / エディタ拡張は未経験
- 情報系の学生・院生、SIer / Web 系企業に就職予定のエンジニア

---

## 構成案（章立て）

### 第1章 — セットアップと最初の一歩
- Claude Code とは何か（Web版との違い、なぜ CLI なのか）
- インストール・認証（`npm i -g @anthropic-ai/claude-code` → `claude`）
- プロジェクトに入って最初に試すコマンド
- CLAUDE.md を作る意味

### 第2章 — 基本的な使い方
- チャット形式でコードを読んでもらう
- ファイル・ディレクトリを丸ごと渡す方法
- `add`, `remove`, `edit` などの基本操作
- 「やっぱりやめる」— 変更の取り消し方

### 第3章 — 実務でよく使うワークフロー
- バグ修正：エラーメッセージを貼って直してもらう
- 新機能追加：仕様を自然言語で伝えて実装してもらう
- リファクタリング：「このファイルを読んで改善案を出して」
- テスト生成：既存コードからユニットテストを作る
- コードレビュー：差分を渡してフィードバックをもらう

### 第4章 — 効果を高えるプロンプトの書き方
- コンテキストを渡す技術（CLAUDE.md の活用）
- 「何をしてほしいか」より「何を達成したいか」を伝える
- ステップに分けて進める vs 一気にやってもらう
- 失敗したときの立て直し方

### 第5章 — スラッシュコマンドとカスタマイズ
- 組み込みスラッシュコマンド一覧（`/commit`, `/review`, `/init` など）
- カスタムスラッシュコマンドを作る
- hooks でワークフローを自動化する
- settings.json でパーミッションをチューニングする

### 第6章 — MCP（Model Context Protocol）連携
- MCP とは何か
- よく使われる MCP サーバー（GitHub, Slack, DB など）
- 独自 MCP サーバーを作る概要

### 第7章 — チーム・プロジェクトでの運用
- CLAUDE.md をチームで育てる
- 権限・セキュリティの考え方（`--dangerously-skip-permissions` を使わないために）
- CI/CD に組み込む（headless モード）
- コスト管理のヒント

## 各章の構成フォーマット

```
## 章タイトル

### この章でできるようになること（冒頭に1〜3行）

### ハンズオン（実際に手を動かせる例）

### よくあるつまずきポイント

### まとめ・次の章へ
```

---

## 制作方針

| 項目 | 方針 |
|---|---|
| 言語 | 日本語（コマンド・コードは英語） |
| サンプルプロジェクト | **TypeScript + Express** による TODO API を全章で使い回す |
| 公開先 | **GitHub リポジトリ**（README を入口に各章 Markdown を配置）|
| 分量 | 1章あたり読了 10〜15 分を目安 |
| 形式 | Markdown（GitHub Pages や Zenn に転用しやすい形） |
| 難易度 | 段階的に上げる（第1〜2章は完全初心者でもOK） |

---

## 優先度・作成順

1. 第1章（セットアップ） — 最優先、ここで躓くと先に進めない
2. 第3章（実務ワークフロー） — 「使える感」を早く掴んでもらう
3. 第2章（基本操作） — 第3章の補足として読んでもらう想定
4. 第4章（プロンプト） — 中級者向け
5. 第5〜7章 — 応用編として後回し可

---

## 決定済み事項

| 項目 | 決定内容 | 理由 |
|---|---|---|
| サンプル言語 | **TypeScript**（Express） | SIer 現場での採用増、Claude Code 自体が Node.js 製で相性が良い |
| 公開先 | **GitHub** | README を入口にし、`docs/` 以下に各章 Markdown を配置 |
| 演習問題 | **各章末に入れる** | 読んだだけで終わらないよう、手を動かせる課題を添える |
